# Incident: Incident: Periodic health check failure
**Date:** 2026-05-18
**Severity:** critical
**Status:** Resolved

## Summary
The application code in app/main.go has a broken /health handler that unconditionally returns HTTP 500 (`w.WriteHeader(http.StatusInternalServerError)`) with body `{"status":"notokay"}`. The root handler `/` also returns HTTP 500 with `{"status":"ded"}`. Kubernetes liveness/readiness probes hit /health, receive 500, mark the pods Unready, and restart them in a loop (4 restarts each, plus prior pods 5gn7s/7tcvp/lq52w/vp7xw). Prometheus scrapes also fail (TargetDown / PlatformmaAppDown). This is a deliberate code bug, not an infrastructure problem — both pods start cleanly (`server listening on :8080`) before the probes fail.

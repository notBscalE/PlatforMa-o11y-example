# Incident: Incident: Periodic health check failure
**Date:** 2026-05-18
**Severity:** high
**Status:** Resolved

## Summary
The /health endpoint in app/main.go is intentionally broken: it explicitly returns HTTP 500 with status 'dead', and it also contains a compile-time bug 'ww.WriteHeader' (undefined variable 'ww' — should be 'w'). The currently running pods (created 2026-05-17) are from an older image and still pass probes, but on the next image build the Go build will fail; if the typo is fixed without correcting the status code, the readinessProbe (httpGet /health) will fail and all replicas will go NotReady. Additionally, the Helm chart helm/platformma-app has no ServiceMonitor, which is why Prometheus reports 'No up metrics found for platformma targets'. The ArgoCD 307 error is a separate client misconfiguration (the health-check client calls http:// but argocd-server now responds with a redirect to https://).

## Immediate Actions Taken
- Do NOT roll/restart the platformma deployment right now — the current pods are running an older image whose /health returns 200; restarting with imagePullPolicy=Always + tag :latest would pull the broken/uncompilable build and take the service down.
- kubectl -n platformma patch deployment platformma-platformma-app -p '{"spec":{"template":{"spec":{"containers":[{"name":"platformma-app","imagePullPolicy":"IfNotPresent"}]}}}}' to prevent a new broken image from being pulled on accidental pod restart until the code fix is merged and a new pinned tag is built.
- Pin image.tag to the last known-good SHA in helm/platformma-app/values.yaml (replace 'latest') and let ArgoCD sync.
- After merging the main.go fix and ServiceMonitor, bump tag to the new build SHA and let ArgoCD roll the deployment.
